# 民族故事问答系统 — 前端联调说明

> 前后端联调时请以此文档为准。如有歧义以实际后端行为为准。

---

## 一、基础信息

| 项目 | 说明 |
|------|------|
| **后端框架** | FastAPI |
| **默认 baseURL** | `http://127.0.0.1:8000`（本地联调） |
| **启动命令** | 项目根目录执行：`uvicorn backend.app:app --reload` |
| **在线文档** | `http://127.0.0.1:8000/docs`（Swagger）、`http://127.0.0.1:8000/redoc`（ReDoc） |
| **Content-Type** | 请求体为 JSON 时请使用 `Content-Type: application/json` |
| **响应格式** | 成功为 JSON 对象/数组；业务错误见各接口说明，部分返回 `{"error": "..."}` |

---

## 二、接口清单

| 序号 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 1 | GET | `/` | 健康检查 |
| 2 | GET | `/stories` | 故事列表（列表项） |
| 3 | GET | `/story/{story_id}` | 故事详情（完整对象） |
| 4 | POST | `/ask` | 基于故事的问答 |

---

## 三、接口详情

### 1. 健康检查

- **Method**：`GET`
- **Path**：`/`
- **请求**：无参数，无 Body
- **成功响应**（200）示例：
```json
{
  "message": "语脉后端启动成功"
}
```

---

### 2. 获取故事列表

- **Method**：`GET`
- **Path**：`/stories`
- **请求**：无参数，无 Body
- **成功响应**（200）：返回**数组**，每项为列表项对象。

**响应类型（TypeScript 示意）：**
```ts
interface StoryListItem {
  id: number;
  title: string;
  ethnic: string;   // 可能为空字符串
  intro: string;   // 可能为空字符串
  summary: string; // 正文前 80 字，不足则全文
}
```

**示例：**
```json
[
  {
    "id": 1,
    "title": "残暴的国王",
    "ethnic": "藏族",
    "intro": "阿古登巴机智惩罚残暴国王",
    "summary": "很久很久以前..."
  },
  {
    "id": 2,
    "title": "妈妈的女儿",
    "ethnic": "未知",
    "intro": "长诗以主人公从出生到婚后的生活轨迹为主线...",
    "summary": "《妈妈的女儿》..."
  }
]
```

- **异常**：数据文件缺失或为空时返回空数组 `[]`，不返回 500。

---

### 3. 获取故事详情

- **Method**：`GET`
- **Path**：`/story/{story_id}`
- **路径参数**：`story_id`（number，整数）
- **成功响应**（200）：返回**完整故事对象**（后端数据中的全部字段）。

**完整故事结构（与 data/stories.json 一致，字段可能缺失时前端需做兜底）：**
```ts
interface Story {
  id: number;
  title: string;
  ethnic: string;
  region?: string;      // 可选
  category?: string;    // 可选
  intro?: string;
  keywords?: string[];  // 可选
  characters?: string[]; // 可选
  content: {
    zh: string;         // 中文正文
    original?: string;   // 民族语原文，可选
  } | string;           // 兼容旧数据：可能直接是字符串
  moral?: string;
  source?: string;
  collector?: string;
}
```

**成功示例：**
```json
{
  "id": 1,
  "title": "残暴的国王",
  "ethnic": "藏族",
  "region": "西藏",
  "category": "民间故事",
  "intro": "阿古登巴机智惩罚残暴国王",
  "keywords": ["阿古登巴", "国王"],
  "characters": ["阿古登巴", "国王"],
  "content": {
    "zh": "很久很久以前，有一个国王...",
    "original": "..."
  },
  "moral": "善恶有报",
  "source": "藏汉双语网",
  "collector": "某某"
}
```

- **错误响应**：`story_id` 不存在时返回**对象**（非数组）：
```json
{
  "error": "story not found"
}
```
前端可通过是否存在 `error` 字段判断。

- **校验**：`story_id` 非整数时由 FastAPI 返回 422，body 为框架默认校验错误格式。

---

### 4. 故事问答

- **Method**：`POST`
- **Path**：`/ask`
- **Headers**：`Content-Type: application/json`
- **请求体：**
```ts
interface AskRequest {
  story_id: number;  // 当前选中的故事 ID
  question: string;  // 用户问题
}
```
示例：
```json
{
  "story_id": 1,
  "question": "这个故事讲了什么？"
}
```

- **成功响应**（200）：
```ts
interface AskResponse {
  answer: string;     // 模型生成的回答
  source: string;     // 主要参考故事标题（第一条来源）
  rag_sources: Array<{ title: string; score: number }>; // 参与回答的故事及相似度
}
```
示例：
```json
{
  "answer": "这个故事讲述了阿古登巴如何用计谋...",
  "source": "残暴的国王",
  "rag_sources": [
    { "title": "残暴的国王", "score": 0.82 },
    { "title": "萨迦格言1", "score": 0.74 },
    { "title": "妈妈的女儿", "score": 0.61 }
  ]
}
```
- `score`：语义相似度（0～1 左右），数值越大越相关；若某条无分数（如当前选中故事被插入）则后端返回 `0`。
- **错误响应**：
  - Body 缺字段或类型错误：FastAPI 返回 **422**，body 为校验错误信息。
  - 服务端异常（如模型调用失败）：返回 **200**，body 为 `{"error": "错误信息字符串"}`。  
  前端建议同时判断：若存在 `error` 字段则按错误处理，并优先展示 `answer` 或 `error`。

---

## 四、前端对接要点

1. **列表页**：请求 `GET /stories`，用返回的 `id`、`title`、`ethnic`、`intro`、`summary` 展示；点击用 `id` 请求详情。
2. **详情页**：请求 `GET /story/{story_id}`，用完整对象渲染；正文优先取 `content.zh`，若 `content` 为字符串则直接使用。
3. **问答**：在选中某故事后请求 `POST /ask`，传 `story_id` 与 `question`；展示 `answer`，可选展示 `rag_sources`（标题 + score）。
4. **错误**：  
   - 详情接口：`response.error === "story not found"` 时提示未找到。  
   - 问答接口：若返回体有 `error`，提示用户或展示 `error` 文案。
5. **兼容**：故事对象中 `region`、`keywords`、`characters`、`moral`、`source`、`collector` 等可能不存在，前端用可选链或默认值避免报错。

## 五、联调检查清单

- [ ] 本地后端已启动（`uvicorn backend.app:app --reload`），可访问 `http://127.0.0.1:8000/docs`。
- [ ] 前端 baseURL 配置为 `http://127.0.0.1:8000`（或当前后端地址）。
- [ ] 跨域：开发环境若前后端不同端口，需后端允许 CORS；当前未显式配置时，可联系后端同学确认或增加 CORS 中间件。
- [ ] 故事列表、详情、问答三条链路均能正常请求并解析 JSON。
- [ ] 对 `GET /story/999`（不存在 id）、`POST /ask` 缺字段等异常情况有错误处理。

## 六、示例请求（curl）

```bash
# 健康检查
curl -X GET "http://127.0.0.1:8000/"

# 故事列表
curl -X GET "http://127.0.0.1:8000/stories"

# 故事详情（将 1 改为实际 id）
curl -X GET "http://127.0.0.1:8000/story/1"

# 问答（Windows PowerShell 下注意转义）
curl -X POST "http://127.0.0.1:8000/ask" -H "Content-Type: application/json" -d "{\"story_id\": 1, \"question\": \"这个故事讲了什么？\"}"
```
