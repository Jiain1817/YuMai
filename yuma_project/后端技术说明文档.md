# 语脉项目后端技术说明文档

## 一、项目当前功能概述

本系统是一个基于 FastAPI 的少数民族故事智能问答平台后端服务，主要实现以下功能：

- **故事管理**：提供故事列表查询、详情查询、搜索功能
- **智能问答**：基于 RAG（检索增强生成）技术，结合 DeepSeek 大模型实现故事相关问答
- **语音服务**：支持文本转语音（TTS）和语音转文本（STT）功能
- **多语言支持**：支持中文、彝文、藏文等多种少数民族语言文本

---

## 二、接口说明

### 1. GET /stories

**功能**：获取故事列表（用于首页展示）

**返回字段说明**：
```json
[
  {
    "id": 1,
    "title": "故事标题",
    "ethnic": "民族名称",
    "intro": "故事简介",
    "summary": "内容摘要（前80字）",
    "cover_image": "/static/images/story1.jpg"
  }
]
```

**字段说明**：
- `id`：故事唯一标识
- `title`：故事标题
- `ethnic`：所属民族
- `intro`：故事简介
- `summary`：自动从正文提取的前80字摘要
- `cover_image`：封面图片路径

---

### 2. GET /story/{story_id}

**功能**：获取故事完整详情（用于详情页展示）

**返回结构**（重点）：
```json
{
  "id": 1,
  "title": "萨迦格言·第1则",
  "ethnic": "藏族",
  "region": "西藏",
  "category": "格言",
  "intro": "智者学识似宝库，积聚珍宝诸格言。",
  "keywords": ["智慧", "格言", "藏族文化"],
  "characters": [],
  "content": {
    "zh": "智者学识似宝库，积聚珍宝诸格言...",
    "yi": "",
    "zang": "མཁས་པ་|ཡོན་ཏན་|མཛོད་འཛིན་|པ།...",
    "original": "མཁས་པ་|ཡོན་ཏན་|མཛོད་འཛིན་|པ།..."
  },
  "moral": "",
  "source": "萨迦格言集",
  "collector": "",
  "cover_image": "/static/images/story2.jpg"
}
```

**content 字段说明**（本次优化重点）：
- `zh`：中文翻译文本（必有）
- `yi`：彝文原文（如果是彝族故事）
- `zang`：藏文原文（如果是藏族故事）
- `original`：原始语言文本（保留字段，与 yi/zang 可能重复）

**注意**：
- 所有字段不存在时返回空字符串 `""`，不会返回 `null`
- 旧数据中的 `ii` 字段会自动映射为 `yi`
- 旧数据中的 `bo` 字段会自动映射为 `zang`

---

### 3. GET /stories/search

**功能**：按标题、民族、关键词搜索故事

**参数**：
- `query`：搜索关键词（URL 参数）

**返回结构**：与 `/stories` 相同

**示例**：
```
GET /stories/search?query=彝族
```

---

### 4. POST /ask

**功能**：基于故事内容的智能问答

**请求体**：
```json
{
  "story_id": 1,
  "question": "这个故事讲了什么？"
}
```

**返回结构**：
```json
{
  "answer": "这个故事讲述了...",
  "source": "萨迦格言·第1则",
  "rag_sources": [
    {"title": "萨迦格言·第1则", "score": 0.95},
    {"title": "萨迦格言·第2则", "score": 0.82}
  ]
}
```

**字段说明**：
- `answer`：AI 生成的回答
- `source`：主要参考来源（最相关的故事标题）
- `rag_sources`：检索到的相关故事列表（含相似度分数）

**技术实现**：
- 使用 FAISS 向量检索找到相关故事
- 使用 DeepSeek 大模型生成回答
- 基于 RAG 技术确保回答基于故事内容

---

### 5. GET /tts/{story_id}

**功能**：将故事中文文本转换为语音

**返回**：音频流（audio/mpeg 格式）

**技术实现**：使用 edge-tts 引擎，中文语音模型

---

### 6. POST /stt

**功能**：将用户上传的音频转换为文本

**请求**：multipart/form-data，字段名 `audio`

**返回**：
```json
{
  "text": "识别出的文本内容"
}
```

**技术实现**：使用 OpenAI Whisper 模型（base 版本）

## 三、数据结构说明（重点）

### content 字段标准化

**优化前的问题**：
- 不同故事的 content 字段结构不统一
- 有的用 `ii`（彝文），有的用 `yi`
- 有的用 `bo`（藏文），有的用 `zang`
- 前端需要写复杂的兼容逻辑

**优化后的方案**：
所有故事的 `content` 字段统一为：
```json
{
  "zh": "中文文本",
  "yi": "彝文文本",
  "zang": "藏文文本"
}
```

**映射规则**：
- `ii` → 自动映射为 `yi`
- `bo` → 自动映射为 `zang`
- `original` → 根据民族自动映射为 `yi` 或 `zang`
- 如果原数据已有 `yi`/`zang`，优先使用
- 不存在的字段返回空字符串 `""`

**设计优势**：
1. **前端解析简单**：只需读取 `content.zh`、`content.yi`、`content.zang`
2. **向后兼容**：保留原有字段（如 `original`），不破坏旧数据
3. **统一规范**：所有接口返回的 content 结构一致

## 六、当前仍存在的问题（客观说明）

1. **返回结构未统一**：
   - 部分接口返回 `{"error": "..."}` 格式
   - 部分接口直接返回数据
   - 建议统一为 `{"code": 0, "msg": "success", "data": {...}}` 格式

2. **数据字段可能缺失**：
   - 部分故事可能缺少 `intro`、`keywords` 等字段
   - 当前通过返回空字符串兼容，但前端需要做空值判断

3. **错误处理不完善**：
   - 部分接口异常时只打印日志，未返回详细错误信息
   - 建议增加统一的异常处理中间件

4. **未做接口版本控制**：
   - 当前所有接口都在根路径下
   - 未来如需升级接口，建议使用 `/api/v1/` 前缀

5. **性能优化空间**：
   - RAG 向量索引在首次调用时构建，可能导致首次请求较慢
   - 建议在服务启动时预加载

## 七、前后端对接说明（关键）

### 1. 列表页对接

**接口**：`GET /stories`

**前端使用**：
```dart
// Flutter 示例
final response = await http.get('http://localhost:8000/stories');
final List stories = jsonDecode(response.body);

// 读取字段
for (var story in stories) {
  print(story['title']);        // 标题
  print(story['ethnic']);       // 民族
  print(story['summary']);      // 摘要
  print(story['cover_image']);  // 封面图
}
```

---

### 2. 详情页对接

**接口**：`GET /story/{story_id}`

**前端使用**：
```dart
// 获取详情
final response = await http.get('http://localhost:8000/story/1');
final story = jsonDecode(response.body);

// 读取中文文本
String zhText = story['content']['zh'];

// 读取彝文文本（如果有）
String yiText = story['content']['yi'];
if (yiText.isEmpty) {
  // 没有彝文
}

// 读取藏文文本（如果有）
String zangText = story['content']['zang'];
if (zangText.isEmpty) {
  // 没有藏文
}

// 其他字段
String title = story['title'];
String ethnic = story['ethnic'];
List keywords = story['keywords'];
```

**重要提示**：
- `content.zh` 一定存在（可能为空字符串）
- `content.yi` 和 `content.zang` 根据民族决定，可能为空
- 前端应该根据 `ethnic` 字段判断显示哪种少数民族语言
