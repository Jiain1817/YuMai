# 后端接口与功能总结

> 面向前端对接与后续开发，基于当前代码实际行为整理。
> 注意：项目包含藏语（如 མཁས་པ་）和彝语字符，属正常数据，非乱码。

---

## 一、项目技术栈

| 类别 | 选型 |
|------|------|
| Web 框架 | FastAPI |
| 语言 | Python 3 |
| 数据存储 | 本地 JSON（`data/stories.json`），无数据库 |
| 向量检索 | sentence-transformers + FAISS（内存懒加载） |
| 大模型 | DeepSeek API（OpenAI 兼容接口） |
| 配置 | `.env` 中配置 `OPENAI_API_KEY` |

启动命令：`uvicorn backend.app:app --reload`
默认地址：`http://127.0.0.1:8000`
在线文档：`http://127.0.0.1:8000/docs`

## 三、接口详情

### 1. GET `/`  健康检查

无参数，无 Body。

响应：
```json
{ "message": "语脉后端启动成功" }
```

---

### 2. GET `/stories`  故事列表

无参数，无 Body。

响应（数组，每项字段）：
```ts
interface StoryListItem {
  id: number;
  title: string;
  ethnic: string;   // 民族，可能为空字符串
  intro: string;    // 简介，可能为空字符串
  summary: string;  // content.zh 前 80 字摘要，不足则全文
}
```

示例：
```json
[
  {
    "id": 1,
    "title": "萨迦格言·第1则",
    "ethnic": "藏族",
    "intro": "智者学识似宝库，积聚珍宝诸格言。",
    "summary": "智者学识似宝库，积聚珍宝诸格言。万水归处为大海，是故百川终汇流"
  }
]
```

异常：数据文件缺失时返回空数组 `[]`，不返回 500。

---

### 3. GET `/story/{story_id}`  故事详情

路径参数：`story_id`（整数）

响应（完整故事对象，字段以 stories.json 为准）：
```ts
interface Story {
  id: number;
  title: string;
  ethnic: string;
  region?: string;
  category?: string;
  intro?: string;
  keywords?: string[];
  characters?: string[];
  content: {
    zh: string;        // 中文正文
    original?: string; // 民族语原文（藏文/彝文等），可选
  } | string;          // 兼容旧数据：可能直接是字符串
  moral?: string;
  source?: string;
  collector?: string;
}
```

错误（story_id 不存在）：
```json
{ "error": "story not found" }
```

story_id 非整数时 FastAPI 返回 422。

---

### 4. POST `/ask`  故事问答

Headers：`Content-Type: application/json`

请求体：
```json
{
  "story_id": 1,
  "question": "这个故事讲了什么？"
}
```

成功响应：
```ts
interface AskResponse {
  answer: string;     // 模型生成的回答
  source: string;     // 主要参考故事标题（RAG 第一条）
  rag_sources: Array<{ title: string; score: number }>; // 参与回答的故事及语义相似度
}
```

示例：
```json
{
  "answer": "这个故事讲述了智者的学识如同宝库...",
  "source": "萨迦格言·第1则",
  "rag_sources": [
    { "title": "萨迦格言·第1则", "score": 0.85 },
    { "title": "萨迦格言·第2则", "score": 0.76 },
    { "title": "萨迦格言·第3则", "score": 0.61 }
  ]
}
```

score 说明：语义相似度（0～1），越大越相关；当前选中故事若不在检索结果中会被强制插入，此时 score 为 0。

错误情况：
- Body 缺字段/类型错误：FastAPI 返回 422
- story_id 不存在：返回 200，body 为 `{"answer": "未找到该故事", "source": "未找到故事", "rag_sources": []}`
- 模型调用失败：返回 200，body 为 `{"error": "错误信息"}`

## 四、数据结构说明

### stories.json 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 唯一标识 |
| title | string | 是 | 故事标题 |
| ethnic | string | 否 | 民族（如"藏族"、"彝族"） |
| region | string | 否 | 地区 |
| category | string | 否 | 分类（如"格言"、"民间故事"） |
| intro | string | 否 | 简介 |
| keywords | string[] | 否 | 关键词 |
| characters | string[] | 否 | 人物 |
| content.zh | string | 推荐 | 中文正文（RAG 检索主要来源） |
| content.original | string | 否 | 民族语原文（藏文/彝文） |
| moral | string | 否 | 寓意 |
| source | string | 否 | 来源 |
| collector | string | 否 | 采集人 |

兼容旧字段：`yi_text`（彝语原文）、`chinese_text`（中文翻译），在无 `content.zh` 时作回退。

## 五、后端内部流程（POST /ask）

```
前端请求 POST /ask
    ↓
app.py → ask_service.answer_question(story_id, question)
    ↓
stories_service.get_story_by_id(story_id)  # 取当前故事
    ↓
agent.ask_model(question, story)
    ├── rag_service.search_similar_stories(question, top_k=3)
    │       # sentence-transformers 编码问题 → FAISS 检索最相似 3 条故事
    │       # 索引首次调用时从 stories.json 懒加载构建，之后复用
    ├── 若当前故事不在检索结果中，强制插入到上下文首位
    ├── 拼接多篇故事上下文（标题 + 民族 + 简介 + 正文）
    └── 调用 DeepSeek API 生成回答
    ↓
返回 { answer, source, rag_sources }
```

## 八、注意事项

1. `app.py` 中有一个重复的 `POST /ask` 定义（与 `routes/ask.py` 中的路由功能相同），存在冗余，见下方清理说明。

4. `.env` 中的 `OPENAI_API_KEY` 实际为 DeepSeek API Key，变量名沿用 OpenAI 兼容命名。
